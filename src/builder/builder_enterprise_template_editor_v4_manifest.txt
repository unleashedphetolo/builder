Builder enterprise template editor update v4

Scope:
- Kept existing builder structure and only updated the editor/target logic needed for template-aligned editing.

Updates:
- Template-backed dynamic editors now render only the fields listed by the template fallback/order metadata.
- Template field order now follows __editor_field_order exactly and does not append unrelated saved keys.
- Internal metadata fields remain hidden from the editor:
  - Editor Field Order
  - Editor Field Labels
  - Editor Fallback Content
- Array/object item collections now infer item fields from the template fallback first, so cards/lists follow the exact template shape.
- Saved title-only or partial items now get missing nested fields restored from the template fallback.
- Local date/time picker support was added for template date/time fields, with a local “Use Local Now” action while still allowing the original display text format.
- Link fields now use a more professional link editor with an open-link action and internal/external URL guidance.
- Section settings now include enterprise section-level controls for:
  - Background colour
  - Text colour
  - Heading colour
  - Accent colour
  - Link colour
  - Body typography
  - Heading typography
  - Type scale
  - Link/button style
- Builder section targets now expose the saved section style as inline style/CSS variables during builder mode so colour/typography edits can be previewed more consistently.

Validation:
- JS/JSX parse check passed across the builder folder.
